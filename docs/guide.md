
# Casebook 사용 가이드

## 🚀 워크플로우
1. 새 케이스 → `open-cases/` 에 생성
2. 조사/분석 진행
3. 해결 완료 → `resolved-cases/<연도>/` 로 이동
4. `update_cases_overview.py` 실행 → 인덱스 최신화
5. PR 생성 → 리뷰 후 main 병합

## 🛠 GitHub Flow
- 브랜치 네이밍: `case/YYYYMMDD-title` 또는 `hotfix/YYYYMMDD-title`
- PR 제목: `case: YYYYMMDD title`
- PR 템플릿 자동 적용
